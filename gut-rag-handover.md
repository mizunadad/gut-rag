## 2. 思想の筋（これが全ての根拠）

### 核心の問い
「腑に落ちる・気配を読む・なんか違う」——これらは脳機能の異なる部位が処理しているから感じる現象であり、言語化の前に身体が先に知っている。

### 3層構造
```
Layer 1：身体・無意識層
　腸神経系(ENS)・基底核・小脳
　→ 直感・Gut never lies

Layer 2：身体‐感情統合層（最強の入口）
　島皮質・扁桃体・上側頭溝
　→ 腑に落ちる・気配を読む・間

Layer 3：統合・意味づけ層
　前頭前皮質・mPFC・DMN
　→ 胆力・人となり・徳
```

### 哲学・仏教との接続
- **般若心経・五蘊（色・受・想・行・識）・十八界**：認識は身体と対象の接触から生まれる。2500年前に「腸で考える」が言語化されていた
- **メルロ＝ポンティ**：身体なき知覚は知覚ではない
- **フランクル**：論証不要の意味の感知
- **龍樹の中論・唯識論**：仏教と哲学は分けられない。同じ筋が東西で独立に発見された
- **グッドハートの法則の身体版**：測ろうとすると逃げる。AIが持てないのはここ

### AIとの差分（Singularity含む）
```
AIが模倣できる：Layer3（言語・論理）→ LLM
接近中：直感・千里眼（ワールドモデル）
部分的：気配・間（アナロジー推論）
未解決：腑に落ちる・Gut never lies（島皮質・ENS）
永遠に残る問い：時間軸上の身体的自己連続性
　　　　　　　　＝チャーマーズの意識のハード問題
```

**結論**：「正しい答え」は出せても「腑に落ちた状態」にはなれない。

## 4. システム設計の核心

### RAGの文書は2種類ある

| 種類 | 内容 | 役割 |
|---|---|---|
| 知識文書 | 般若心経・哲学・神経科学 | 「なぜそう感じたか」の解釈基盤 |
| 経験文書 | gut-loggerの臨場感ログ | 「自分はどう感じてきたか」のパターン |

**両方必要。** 経験に知識が乗ることで意味が生まれる。

### 担当制という天才的設計
```
「担当」という役割が同時に：
・入力フィルター（何を書くかの境界）
・動機づけ（自分の領域への責任感）
・重要度の一次判定（担当が動いた＝重要）

→ 役割が身体知の記録装置になる
→ 選定プロセスのオーバーヘッドがゼロ
```

### 文脈×感覚のパターンが「自らナッジ」の正体
```
「サプライヤー訪問×ヒヤッと」が当たり続ける
　↓
自分の腸はこの文脈に敏感だとわかる
　↓
次回同じ文脈で自然とアンテナが立つ
```

## 5. 作成済みアーティファクト

### ① gut-brain-map.html
**役割**：10の身体知の言葉と脳機能の対応を示す知識地図  
**操作**：Layerフィルター・カードタップで詳細展開  
**位置づけ**：知識文書の視覚的入口

### ② gut-socratic.html
**役割**：絵文字タップで自分の感覚を探る「問いかけツール」  
**フロー**：人/場所/もの/ココロ → 絵文字 → 選択肢 → 「もっと深く」でLLM呼び出し  
**コスト設計**：LLMは最後の1回だけ（課金最小化）  
**位置づけ**：探索・言語化ツール

### ③ gut-logger-v2.html（最新版）
**役割**：「なんか」をその場で残す記録ツール＋パターン可視化  
**フロー**：
```
Step1：感覚（絵文字タップ）
Step2：文脈（場面タップ）
Step3：言語化サポート（感覚ごとの問い＋選択肢）← 最新追加
Step4：一言（任意・空欄OK）
```
**パターンタブ**：文脈×感覚マトリクス・当たり率・上位パターンのインサイト  
**位置づけ**：経験文書の生成装置

### ④ gut-rag-graph-v2.html
**役割**：全体の知識構造をインタラクティブに可視化  
**タブ**：関係図 / required→enablesフロー / Singularity対比表  
**位置づけ**：プロジェクト全体の地図

## 6. 設計思想のキーワード（次セッションで使う言葉）

| 言葉 | 意味 |
|---|---|
| 腸で考える | 言語化以前の身体知。Layer1〜2の処理 |
| 腑に落ちる | 論理と身体感覚が一致した状態。島皮質 |
| 臨場感 | 記録に感情フラグ・文脈・結果が揃った状態 |
| ふとした瞬間 | 直感と記録の間。言語化サポートが必要な場所 |
| 当たり率 | 直感の精度。蓄積することで腸への信頼が生まれる |
| 担当制 | 役割が自然な記録を生む。選定プロセス不要 |
| 常在 | 習慣化ではなくホーム画面に置く。射程に置く設計 |
| required | この概念を理解するために必須の前提 |
| enables | これがあると次に進める条件 |
